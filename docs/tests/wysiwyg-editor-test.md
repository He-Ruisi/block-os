# 所见即所得编辑器测试指南

## 测试目标
验证编辑器区域的 Markdown 渲染是否达到所见即所得的效果，与 AI 对话区域的渲染保持一致。

## 已实现的功能

### 1. 扩展功能
- ✅ 代码块语法高亮（使用 lowlight）
- ✅ 表格支持
- ✅ 任务列表
- ✅ 排版增强（智能引号、破折号）
- ✅ 占位符提示

### 2. 样式渲染
所有样式参考 `MarkdownRenderer` 组件，确保一致性：

- ✅ 标题（H1-H6）- 带边框、间距、字号
- ✅ 段落 - 行高、间距
- ✅ 引用块 - 紫色左边框、浅紫背景
- ✅ 列表（有序/无序）- 缩进、间距
- ✅ 行内代码 - 红色文字、灰色背景、边框
- ✅ 代码块 - 语法高亮、边框、圆角
- ✅ 表格 - 边框、悬停效果、表头样式
- ✅ 任务列表 - 复选框、删除线
- ✅ 链接 - 紫色、悬停下划线
- ✅ 分隔线 - 双线边框
- ✅ 强调（加粗、斜体、删除线）

### 3. 工具栏增强
- ✅ 添加表格插入按钮
- ✅ 添加任务列表按钮
- ✅ 保留原有的所有格式按钮

## 测试步骤

### 基础测试
1. 启动开发服务器：`bun run dev`
2. 打开编辑器，查看欢迎页面是否正确渲染
3. 测试各种 Markdown 语法：

#### 标题测试
```
# 一级标题
## 二级标题
### 三级标题
```

#### 文本格式测试
```
**加粗文字**
*斜体文字*
~~删除线~~
`行内代码`
```

#### 列表测试
```
- 无序列表项 1
- 无序列表项 2
  - 嵌套项

1. 有序列表项 1
2. 有序列表项 2
```

#### 任务列表测试
```
- [ ] 未完成任务
- [x] 已完成任务
```

#### 引用测试
```
> 这是一段引用文字
> 可以有多行
```

#### 代码块测试
````
```javascript
function hello() {
  console.log('Hello, World!')
}
```
````

#### 表格测试
使用工具栏的表格按钮插入 3x3 表格，验证：
- 表头样式（灰色背景）
- 单元格边框
- 悬停效果

### 对比测试
1. 在 AI 对话区域输入相同的 Markdown 内容
2. 对比编辑器区域和 AI 对话区域的渲染效果
3. 确保两者样式一致

### 交互测试
1. 测试工具栏按钮是否正常工作
2. 测试快捷键（Ctrl+B 加粗、Ctrl+I 斜体等）
3. 测试 BubbleMenu 是否正常显示
4. 测试拖拽 AI 内容到编辑器

## 预期结果
- 所有 Markdown 语法都应该实时渲染为所见即所得的效果
- 编辑器样式与 AI 对话区域保持一致
- 代码块有语法高亮
- 表格和任务列表正常显示和交互
- 无样式冲突或显示异常

## 已知限制
- 代码块语法高亮使用 lowlight，支持常见语言
- 表格可调整大小（resizable: true）
- 任务列表支持嵌套
