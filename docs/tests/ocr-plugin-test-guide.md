# OCR 插件测试指南

## 测试目标
验证 OCR 插件系统的完整功能，包括插件管理、OCR 识别、编辑器集成和 Block 保存。

## 前置条件
- 应用已启动（`bun run dev`）
- 已登录用户账号
- 浏览器支持摄像头访问（HTTPS 或 localhost）

## 测试步骤

### 1. 插件列表查看
**操作**：
1. 点击左侧 ActivityBar 的"插件"图标（🧩 Puzzle）
2. 查看插件列表

**预期结果**：
- ✅ 显示"已安装插件"标题
- ✅ 显示 OCR 插件卡片
  - 插件名称：OCR 文字识别
  - 插件描述：使用 PaddleOCR 识别图片中的文字，支持摄像头拍照和图片上传
  - 插件状态：active（绿色）
  - 插件版本：v1.0.0
- ✅ 显示操作按钮：打开、设置、卸载

---

### 2. 打开 OCR 插件
**操作**：
1. 点击 OCR 插件卡片上的"打开"按钮

**预期结果**：
- ✅ 插件列表消失，显示 OCR 插件 UI
- ✅ 顶部显示"OCR 文字识别"标题和关闭按钮（X）
- ✅ 显示预览区域（灰色背景，提示"点击下方按钮开始"）
- ✅ 显示两个按钮："开启摄像头"和"上传图片"

---

### 3. 摄像头拍照识别
**操作**：
1. 点击"开启摄像头"按钮
2. 允许浏览器访问摄像头
3. 对准文字内容
4. 点击"拍照"按钮
5. 点击"识别文字"按钮

**预期结果**：
- ✅ 摄像头画面显示在预览区域
- ✅ 拍照后显示静态图片预览
- ✅ 按钮变为"识别文字"和"重置"
- ✅ 点击识别后显示"识别中..."
- ✅ 识别完成后显示识别结果文本
- ✅ 显示"插入编辑器"和"保存为 Block"按钮

---

### 4. 图片上传识别
**操作**：
1. 点击"重置"按钮（如果之前有操作）
2. 点击"上传图片"按钮
3. 选择一张包含文字的图片
4. 点击"识别文字"按钮

**预期结果**：
- ✅ 图片预览显示在预览区域
- ✅ 识别过程和结果显示正常
- ✅ 识别结果准确（根据图片内容）

---

### 5. 插入编辑器
**操作**：
1. 完成识别后，点击"插入编辑器"按钮
2. 切换到编辑器视图（如果在 AI 沉浸模式，先切换到混合模式）

**预期结果**：
- ✅ 显示成功提示："已插入编辑器"
- ✅ 编辑器中出现 SourceBlock（橙色边框）
- ✅ SourceBlock 标签显示"📷 OCR 识别"
- ✅ SourceBlock 内容为识别的文字
- ✅ 文字格式正确（段落分隔、换行保留）

---

### 6. 保存为 Block
**操作**：
1. 完成识别后，点击"保存为 Block"按钮
2. 打开 Block 空间（右侧面板 → Block 标签页）

**预期结果**：
- ✅ 显示成功提示："已保存为 Block"
- ✅ Block 空间中出现新的 Block
- ✅ Block 标题：OCR 识别结果
- ✅ Block 标签：OCR、图片识别
- ✅ Block 内容为识别的文字
- ✅ Block 来源类型：import

---

### 7. 插件设置
**操作**：
1. 返回插件列表（点击关闭按钮）
2. 点击 OCR 插件的"设置"按钮（齿轮图标）
3. 查看和修改配置

**预期结果**：
- ✅ 显示"OCR API 配置"标题
- ✅ 显示两个输入框：
  - API 地址（默认值：https://lao3t2m4beyceb6c.aistudio-app.com/layout-parsing）
  - API Token（默认值：74fc1211d4321e9438158dae3d22f8005fd5e4e2）
- ✅ 显示"保存配置"按钮
- ✅ 修改配置后点击保存，显示成功提示："配置已保存"
- ✅ 配置持久化（刷新页面后配置仍然存在）

---

### 8. 插件卸载
**操作**：
1. 返回插件列表
2. 点击 OCR 插件的"卸载"按钮（垃圾桶图标）
3. 确认卸载

**预期结果**：
- ✅ 显示确认对话框："确定要卸载此插件吗？"
- ✅ 确认后插件从列表中消失
- ✅ 显示"暂无插件"占位符
- ✅ 插件配置被清理（localStorage 中 `plugin:ocr-plugin:*` 键被删除）

---

### 9. 错误处理测试
**操作**：
1. 修改 API 配置为无效地址
2. 尝试识别图片

**预期结果**：
- ✅ 显示错误提示（红色背景）
- ✅ 错误信息清晰（例如："HTTP 404 Not Found"）
- ✅ 应用不崩溃，可以继续操作

## 性能测试

### 识别速度
- 小图片（< 1MB）：< 3 秒
- 中等图片（1-3MB）：3-5 秒
- 大图片（> 3MB）：5-10 秒

### 内存占用
- 插件加载后内存增加：< 10MB
- 识别过程中内存峰值：< 50MB

## 已知问题
- 摄像头权限被拒绝后，需要刷新页面重新请求
- 识别超大图片（> 10MB）可能导致浏览器卡顿
- 部分浏览器不支持 `facingMode: 'environment'`（后置摄像头）
