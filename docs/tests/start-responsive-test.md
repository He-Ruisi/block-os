# 开始响应式测试

## 准备工作

### 1. 启动开发服务器

```bash
cd block-os
bun run dev
```

服务器启动后，你会看到：

```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

### 2. 选择测试方式

#### 方式 A: 浏览器开发者工具测试（推荐新手）

**优点**: 快速、无需真机、可测试多种设备
**缺点**: 无法测试真实触摸交互

**步骤**:
1. 在桌面浏览器打开 `http://localhost:5173`
2. 按 `F12` 打开开发者工具
3. 点击"Toggle device toolbar"（或按 `Ctrl+Shift+M`）
4. 选择设备：iPad Pro / iPhone 14 Pro Max 等
5. 按照 [响应式快速测试指南](./responsive-quick-test.md) 进行测试

#### 方式 B: 真机测试（推荐完整测试）

**优点**: 真实触摸体验、真实性能表现
**缺点**: 需要真机、需要局域网配置

**步骤**:
1. 确保电脑和移动设备在同一 WiFi
2. 查看开发服务器的 Network 地址（如 `http://192.168.1.100:5173`）
3. 在移动设备浏览器访问该地址
4. 按照 [响应式测试检查清单](./responsive-checklist.md) 进行测试

#### 方式 C: 使用 ngrok（外网测试）

**优点**: 可在任何网络测试、可分享给他人
**缺点**: 需要安装 ngrok、有速度限制

**步骤**:
1. 安装 ngrok: https://ngrok.com/download
2. 启动隧道: `ngrok http 5173`
3. 复制 ngrok 提供的 HTTPS URL
4. 在任何设备访问该 URL

## 测试流程

### 第一步：浏览器模拟测试（30分钟）

使用 Chrome DevTools 测试以下设备：

1. **iPad Pro (1024×1366)** - 平板竖屏
2. **iPad Pro (1366×1024)** - 平板横屏
3. **iPhone 14 Pro Max (430×932)** - 手机竖屏
4. **iPhone 14 Pro Max (932×430)** - 手机横屏

**测试内容**:
- [ ] 布局响应式（ActivityBar / Sidebar / Editor / RightPanel）
- [ ] AI 对话主路径（问候 → 发送 → 写入编辑器）
- [ ] 项目和文档管理（新建 / 打开 / 重命名 / 删除）
- [ ] Block 捕获功能（捕获 → 查看 → 搜索）

**参考文档**: [响应式快速测试指南](./responsive-quick-test.md)

### 第二步：iPad 真机测试（30分钟）

在真实 iPad 上测试：

**竖屏测试**:
- [ ] 布局适配（浮层模式）
- [ ] 触摸目标尺寸（≥ 44px）
- [ ] 滑动手势（关闭 Sidebar / RightPanel）
- [ ] 长按菜单（文档操作）
- [ ] 滚动流畅度

**横屏测试**:
- [ ] 布局适配（与竖屏一致）
- [ ] 横竖屏切换流畅
- [ ] 功能正常工作

**参考文档**: [响应式测试检查清单](./responsive-checklist.md)

### 第三步：iPhone 真机测试（30分钟）

在真实 iPhone 上测试：

**竖屏测试**:
- [ ] 底部导航栏（ActivityBar）
- [ ] 全宽浮层（Sidebar / RightPanel）
- [ ] iOS Safe Area 适配
- [ ] 触摸交互流畅
- [ ] 字号清晰可读（17px）

**横屏测试**:
- [ ] 布局适配
- [ ] 垂直间距优化
- [ ] 横竖屏切换流畅

**参考文档**: [响应式测试检查清单](./responsive-checklist.md)

### 第四步：自动化测试（5分钟）

在浏览器控制台运行自动化测试脚本：

1. 打开 `http://localhost:5173`
2. 按 `F12` 打开控制台
3. 复制 [responsive-test-script.js](./responsive-test-script.js) 内容
4. 粘贴到控制台并运行
5. 查看测试结果

### 第五步：填写测试报告（10分钟）

根据测试结果填写：

1. **[响应式测试矩阵](./responsive-test-matrix.md)** - 完整测试矩阵
2. **[响应式测试检查清单](./responsive-checklist.md)** - 详细检查清单

## 测试重点

### 必测项（P0）

这些是最重要的功能，必须全部通过：

1. **AI 对话主路径**
   - 沉浸式模式 → 发送消息 → 写入编辑器 → 自动创建文档

2. **布局响应式**
   - 桌面 / 平板 / 手机三种模式布局正确

3. **触摸交互**
   - 所有按钮触摸目标 ≥ 44px
   - 滑动手势关闭 Sidebar / RightPanel

4. **数据持久化**
   - 刷新页面数据保持
   - 关闭重开数据保持

### 重要项（P1）

这些功能应该正常工作，但可以有小问题：

1. **项目和文档管理**
2. **Block 捕获和搜索**
3. **主题切换**
4. **横竖屏切换**

### 次要项（P2）

这些功能可以在后续优化：

1. **性能优化**
2. **动画流畅度**
3. **错误提示**

## 常见问题

### Q: 移动设备无法访问开发服务器

**A**: 检查以下几点：
1. 电脑和移动设备在同一 WiFi
2. 电脑防火墙允许 5173 端口
3. 使用 Network 地址而不是 localhost
4. 尝试使用 ngrok

### Q: 触摸手势不响应

**A**: 
1. 确保在真机上测试（模拟器可能不支持）
2. 检查浏览器是否支持 Touch Events
3. 在 Safari 浏览器测试（iOS 推荐）

### Q: 布局显示异常

**A**:
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 硬刷新页面（Ctrl+Shift+R）
3. 检查响应式样式是否加载

### Q: IndexedDB 不工作

**A**:
1. 检查浏览器是否支持 IndexedDB
2. 清除浏览器数据和缓存
3. 检查浏览器隐私设置
