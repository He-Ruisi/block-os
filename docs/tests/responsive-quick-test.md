# BlockOS 响应式快速测试指南

## 快速开始

### 1. 启动开发服务器

```bash
cd block-os
bun run dev
```

服务器启动后，访问 `http://localhost:5173`

### 2. 浏览器开发者工具测试（5分钟快速验证）

#### Chrome/Edge DevTools

1. 按 `F12` 打开开发者工具
2. 点击"Toggle device toolbar"图标（或按 `Ctrl+Shift+M`）
3. 选择设备模拟：

**快速测试序列**:

```
iPad Pro (1024×1366) → 测试平板竖屏
iPad Pro (1366×1024) → 测试平板横屏
iPhone 14 Pro Max (430×932) → 测试手机竖屏
iPhone 14 Pro Max (932×430) → 测试手机横屏
```

#### 每个设备测试 3 个核心路径

**路径 1: AI 对话 → 写入编辑器**
1. 看到 AI 沉浸式模式（大号问候语）
2. 输入"你好，介绍一下你自己"
3. 点击发送，等待 AI 回复
4. 点击"写入编辑器"按钮
5. 验证：自动切换到混合模式，创建新文档，AI 内容写入编辑器

**路径 2: 侧边栏和项目管理**
1. 点击左侧 ActivityBar 的"文件"图标
2. 验证：Sidebar 以浮层形式弹出（平板/手机）
3. 点击"新建项目"
4. 输入项目名称"测试项目"
5. 点击项目展开文档列表

**路径 3: Block 捕获**
1. 在 AI 对话中发送消息
2. 等待 AI 回复
3. 点击"◆ 捕获为Block"按钮
4. 验证：显示成功提示，自动切换到"Block空间"标签页
5. 在 Block 列表中看到新捕获的 Block

### 3. 真机测试（iPad/iPhone）

#### 方法 A: 局域网访问（推荐）

1. 确保电脑和移动设备在同一 WiFi 网络
2. 查看电脑 IP 地址：
   ```bash
   # Windows
   ipconfig
   
   # macOS/Linux
   ifconfig
   ```
3. 在移动设备浏览器访问：`http://<电脑IP>:5173`
   - 例如：`http://192.168.1.100:5173`

#### 方法 B: 使用 ngrok（外网访问）

1. 安装 ngrok：https://ngrok.com/download
2. 启动隧道：
   ```bash
   ngrok http 5173
   ```
3. 复制 ngrok 提供的 HTTPS URL
4. 在移动设备浏览器访问该 URL

## 常见问题排查

### Q: 移动设备无法访问开发服务器

**解决方案**:
1. 确保电脑和移动设备在同一 WiFi
2. 检查电脑防火墙是否阻止 5173 端口
3. 尝试使用 ngrok 外网访问

### Q: 响应式布局不生效

**解决方案**:
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 硬刷新页面（Ctrl+Shift+R）
3. 检查浏览器是否支持 CSS 媒体查询

### Q: 触摸手势不响应

**解决方案**:
1. 确保在真机上测试（模拟器可能不支持）
2. 检查浏览器是否支持 Touch Events
3. 尝试在 Safari 浏览器测试（iOS）

### Q: IndexedDB 不工作

**解决方案**:
1. 检查浏览器是否支持 IndexedDB
2. 清除浏览器数据和缓存
3. 检查浏览器隐私设置（是否阻止存储）
