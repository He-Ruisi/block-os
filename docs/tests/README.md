### 详细步骤

1. **准备阶段**（10分钟）
   - 阅读 [开始响应式测试](./start-responsive-test.md)
   - 启动开发服务器
   - 准备测试设备

2. **浏览器模拟测试**（30分钟）
   - 使用 [响应式快速测试](./responsive-quick-test.md)
   - 测试 iPad Pro / iPhone 14 Pro Max 等设备
   - 验证布局响应式和核心功能

3. **真机测试**（1小时）
   - 使用 [响应式测试检查清单](./responsive-checklist.md)
   - iPad 测试（竖屏/横屏）
   - iPhone 测试（竖屏/横屏）
   - 触摸交互和手势测试

4. **自动化验证**（5分钟）
   - 运行 [响应式测试脚本](./responsive-test-script.js)
   - 验证布局和样式

5. **报告提交**（10分钟）
   - 填写 [测试报告模板](./test-report-template.md)
   - 更新 [测试进度跟踪](./test-progress.md)
   - 提交测试结果

## 📊 测试优先级

### P0 - 必测项（阻塞发布）

这些功能必须全部通过，否则不能发布：

- ✅ **AI 对话主路径**
  - 沉浸式模式 → 发送消息 → 写入编辑器 → 自动创建文档
  
- ✅ **布局响应式**
  - 桌面模式（≥ 1280px）
  - 平板模式（768px - 1280px）
  - 手机模式（< 768px）
  
- ✅ **触摸交互**
  - 触摸目标 ≥ 44px
  - 滑动手势关闭 Sidebar / RightPanel
  
- ✅ **数据持久化**
  - 刷新页面数据保持
  - 关闭重开数据保持

### P1 - 重要项（影响体验）

这些功能应该正常工作，但可以有小问题：

- 📝 项目和文档管理
- 📝 Block 捕获和搜索
- 📝 主题切换
- 📝 横竖屏切换

### P2 - 次要项（后续优化）

这些功能可以在后续版本优化：

- 🔧 性能优化
- 🔧 动画流畅度
- 🔧 错误提示

## ❓ 常见问题

### Q: 移动设备无法访问开发服务器？

**A**: 
1. 确保电脑和移动设备在同一 WiFi
2. 检查电脑防火墙是否阻止 5173 端口
3. 使用 Network 地址而不是 localhost
4. 尝试使用 ngrok

### Q: 触摸手势不响应？

**A**:
1. 确保在真机上测试（模拟器可能不支持）
2. 检查浏览器是否支持 Touch Events
3. 在 Safari 浏览器测试（iOS 推荐）

### Q: 布局显示异常？

**A**:
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 硬刷新页面（Ctrl+Shift+R）
3. 检查响应式样式是否加载

### Q: 如何提交测试报告？

**A**:
1. 填写 [测试报告模板](./test-report-template.md)
2. 更新 [测试进度跟踪](./test-progress.md)
3. 提交到项目文档目录
