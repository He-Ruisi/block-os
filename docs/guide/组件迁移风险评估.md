## ⚠️ 风险等级说明

### 🟢 低风险
- 组件逻辑简单
- 样式独立，不影响其他模块
- 容易测试和验证
- 回滚成本低

**示例**：StarredView、SearchView、Button、Card

---

### 🟡 中风险
- 组件有复杂交互（拖拽、搜索、定位）
- 涉及全局状态或多组件联动
- 需要多场景测试
- 回滚成本中等

**示例**：Dialog、Popover、Table、Calendar

---

### 🔴 高风险
- 核心业务组件
- 涉及数据持久化或网络请求
- 影响多个模块
- 回滚成本高

**示例**：Editor（TipTap）、AI 对话核心逻辑

---

## 💡 修改建议

### 1. 渐进式重构
- ✅ 一次只重构一个组件
- ✅ 每次重构后立即测试
- ✅ 保留旧 CSS 文件直到验证通过
- ✅ Git 提交每个组件的修改

### 2. 测试策略
- **低风险组件**：快速视觉检查 + 基础功能测试
- **中风险组件**：完整功能测试 + 边界情况测试
- **高风险组件**：全面回归测试 + 用户验收测试

### 3. 回滚计划
- 每次修改前 Git 提交
- 保留旧 CSS 文件作为备份
- 遇到问题立即回滚，不要强行修复
