## 🎨 核心特性

### 1. 自动扫描历史问题 🔍

**功能**：修复前自动检查 bugs.md

**实现**：
```
1. 读取 bugs.md 文件
2. 搜索相关关键词
3. 匹配错误信息、组件名、症状
4. 返回相似问题列表
```

**价值**：
- 避免重复劳动
- 参考之前的解决方案
- 发现关联问题

### 2. 标准化记录格式 📝

**格式**（≤ 400 字）：
```markdown
## #序号 - 标题 (日期) 🔴/🟡/🟢

**问题描述**:
- 现象、触发、影响

**根本原因**:
[1-2 句话]

**解决方案**:
- 修改文件、核心改动、预防措施

**相关问题**: [#序号]
```

**特点**：
- 结构化：便于查找和理解
- 简洁：严格控制 400 字
- 完整：包含诊断和解决方案
- 可追溯：支持关联问题

### 3. 分类管理系统 🗂️

**按功能分类**：
- 编辑器相关
- AI 对话相关
- Block 系统相关
- 性能问题
- UI/样式问题

**按严重程度**：
- 🔴 严重：功能完全不可用
- 🟡 中等：影响部分功能
- 🟢 轻微：小问题或优化

### 4. 智能工作流程 🔄

```
用户报告 Bug
    ↓
激活 Fix Skill (#fix)
    ↓
1. 扫描 bugs.md 🔍
   - 查找相似问题
   - 参考历史方案
    ↓
2. 诊断问题 🔬
   - 收集错误信息
   - 定位错误源头
   - 分析根本原因
    ↓
3. 实施修复 🔧
   - 修改代码
   - 运行类型检查
   - 验证修复效果
    ↓
4. 记录知识库 📝
   - 按格式记录到 bugs.md
   - 控制在 400 字以内
   - 添加分类和标记
    ↓
5. 报告结果 ✅
   - 向用户报告修复情况
   - 提供 bug 编号
```

## 🎓 使用示例

### 示例 1：简单的空指针错误

**用户报告**：
```
我遇到了一个 bug：
编辑器打不开，控制台显示 "Cannot read property 'schema' of null"
```

**Fix Skill 处理**：
```
1. ✅ 扫描 bugs.md - 未找到相似问题
2. 🔬 诊断：useEditor hook 返回 null
3. 🔧 修复：添加空值检查
4. 📝 记录到 bugs.md (#001)
5. ✅ 报告：Bug 已修复
```

**记录**（180 字）：
```markdown
## #001 - TipTap 编辑器初始化失败 (2026-04-09) 🔴

**问题描述**:
- 现象：编辑器区域空白，控制台报错
- 触发：页面首次加载时
- 影响：无法使用编辑器功能

**根本原因**:
useEditor hook 返回 null 导致空指针异常。

**解决方案**:
- 修改文件：src/components/Editor.tsx
- 核心改动：添加 `if (!editor) return null`
- 预防措施：所有访问 editor 的地方先检查

**相关问题**: 无
```

### 示例 2：发现相似问题

```
AI 对话框点击发送没反应

```
