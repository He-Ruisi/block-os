# Bug 修复日志 - 2026-04-09 17:30

## 修复的 Bug

### Bug 1: 捕获 Block 对话框无响应 ❌ → ✅

**问题描述**：
- 用户填写标题后点击"捕获"按钮无反应
- 原因：标签输入框的 `onBlur` 事件与"捕获"按钮点击冲突

**问题分析**：
```typescript
// 原代码
<input
  onBlur={handleAddTag}  // ❌ 点击捕获按钮时先触发 blur
  // ...
/>
```

当用户在标签输入框中输入内容后，直接点击"捕获"按钮时：
1. 输入框失去焦点，触发 `onBlur` 事件
2. `handleAddTag` 执行，可能导致组件重新渲染
3. "捕获"按钮的点击事件被中断或延迟
4. 用户感觉按钮无响应

**解决方案**：
1. 移除 `onBlur` 事件
2. 添加独立的"+ 添加标签"按钮
3. 保留 Enter 键快捷添加功能

```typescript
// 修复后
<input
  onKeyDown={handleKeyDown}  // ✅ Enter 键添加
  // 移除 onBlur
/>
{tagInput.trim() && (
  <button onClick={handleAddTag}>  // ✅ 独立按钮
    + 添加标签
  </button>
)}
```

**用户体验改进**：
- ✅ 点击"捕获"按钮立即响应
- ✅ Enter 键快速添加标签
- ✅ 独立按钮明确操作意图
- ✅ 避免意外触发

### Bug 2: AI 内容写入编辑器不识别 Markdown ❌ → ✅

**问题描述**：
- AI 输出的 Markdown 格式内容（如 `### 标题`）直接显示为纯文本
- 用户看到 `###` 等符号，而不是格式化的标题

**问题分析**：
```typescript
// 原代码
editor.chain().focus().insertContent(content).run()
// ❌ 直接插入纯文本，不解析 Markdown
```

TipTap 的 `insertContent` 默认插入纯文本，不会自动解析 Markdown 语法。

**解决方案**：
实现 Markdown 到 HTML 的转换逻辑：

```typescript
const handleInsertAIContent = (content: string) => {
  const lines = content.split('\n')
  
  lines.forEach((line) => {
    // 标题：### 文本 → <h3>文本</h3>
    if (line.match(/^#{1,6}\s/)) {
      const level = line.match(/^(#{1,6})/)?.[1].length || 1
      const text = line.replace(/^#{1,6}\s+/, '')
      editor.chain().insertContent(`<h${level}>${text}</h${level}>`).run()
    }
    // 无序列表：- 文本 → <ul><li>文本</li></ul>
    else if (line.match(/^[-*]\s+/)) {
      const text = line.replace(/^[-*]\s+/, '')
      editor.chain().insertContent(`<ul><li><p>${text}</p></li></ul>`).run()
    }
    // 有序列表：1. 文本 → <ol><li>文本</li></ol>
    else if (line.match(/^\d+\.\s+/)) {
      const text = line.replace(/^\d+\.\s+/, '')
      editor.chain().insertContent(`<ol><li><p>${text}</p></li></ol>`).run()
    }
    // 普通段落，处理行内格式
    else {
      let html = line
      // **加粗** → <strong>加粗</strong>
      html = html.replace(/\*\*(.+?)\*\*/g, '<strong>$1</strong>')
      // *斜体* → <em>斜体</em>
      html = html.replace(/\*(.+?)\*/g, '<em>$1</em>')
      // `代码` → <code>代码</code>
      html = html.replace(/`(.+?)`/g, '<code>$1</code>')
      
      editor.chain().insertContent(`<p>${html}</p>`).run()
    }
  })
}
```

**支持的 Markdown 语法**：
- ✅ 标题：`# H1` ~ `###### H6`
- ✅ 加粗：`**文本**` 或 `__文本__`
- ✅ 斜体：`*文本*` 或 `_文本_`
- ✅ 行内代码：`` `代码` ``
- ✅ 无序列表：`- 项目` 或 `* 项目`
- ✅ 有序列表：`1. 项目`
- ✅ 空行：自动插入段落

**转换示例**：

输入（AI 输出）：
```markdown
### 产品特点

这是一个**强大**的知识管理系统，支持：

- 双向链接
- 块引用
- AI 辅助写作

使用 `[[]]` 创建链接。
```

输出（编辑器显示）：
```
标题3: 产品特点

这是一个【加粗】强大【/加粗】的知识管理系统，支持：

• 双向链接
• 块引用
• AI 辅助写作

使用【代码】[[]]【/代码】创建链接。
```

1. **逐行处理**：避免复杂的正则表达式
2. **优先级匹配**：标题 > 列表 > 段落
3. **行内格式**：在段落中处理加粗、斜体、代码
4. **HTML 插入**：使用 TipTap 的 HTML 解析能力

### 性能考虑
- 正则表达式简单高效
