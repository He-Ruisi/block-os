# 一套可直接用的 AI 规则

你可以直接复制这段，作为项目规则。

---

## **UI 实现规则 / AI Coding Rules**

### 1. 总体原则
- 本项目 UI 层统一使用 **shadcn/ui + Tailwind CSS**
- 禁止直接复制 v0 导出的完整页面代码到业务组件中
- 禁止在已有复杂业务组件中直接混入大量 Tailwind className 改样式
- 所有界面美化必须通过“展示组件”完成，不直接改动核心逻辑组件结构

---

### 2. 组件分层规则
- 业务逻辑、状态管理、拖拽、编辑器联动放在 **Container 组件**
- 纯视觉结构放在 **View / Presentational 组件**
- Container 只负责传递 props 和处理事件
- View 组件不直接访问业务 store、不发请求、不处理复杂状态逻辑

---

### 3. shadcn/ui 使用规则
- 优先使用 shadcn/ui 现有组件：Button、Card、Input、Dialog、Popover、DropdownMenu、Tabs、Sheet、ScrollArea、Tooltip、Badge、Separator
- 若 v0 生成的界面使用了原生 div/button/input，必须优先替换为 shadcn/ui 对应组件
- 不允许为了还原 v0 页面而绕过 shadcn/ui 重新手写一套按钮、卡片、输入框基础样式

---

### 4. Tailwind className 规则
- 允许在展示组件中使用 Tailwind
- 禁止在 JSX 中堆积超长 className，超过 5~8 个 utility 时必须提取
- 重复出现的样式必须抽成：
  - `cn()` 组合
  - 公共 UI 组件
  - `cva()` 变体
- 禁止把 v0 导出的超长 className 原样保留在多个文件中

---

### 5. 样式复用规则
- 颜色、圆角、间距、阴影统一使用设计 token
- 所有交互状态（selected / active / dragging / editing）必须通过统一变体控制
- 重复卡片样式抽为 `BlockCardShell`
- 重复工具栏样式抽为 `PanelToolbar`
- 重复输入样式抽为 `SearchInput` 或基于 shadcn Input 封装

---

### 6. 修改 existing code 规则
- 不修改拖拽逻辑
- 不修改 Tiptap 联动逻辑
- 不修改 block 数据结构
- 只允许替换最外层 DOM 结构为 shadcn/ui 外壳
- 所有逻辑变更必须单独说明原因

---

### 7. v0 使用规则
- v0 仅作为视觉参考和布局参考
- 不直接照搬 v0 生成代码
- 必须将 v0 代码“翻译”为本项目的 shadcn/ui 风格组件
- 优先提取为可复用组件，而不是复制页面片段

---

### 8. 输出要求
- 输出代码时优先给出：
  1. 新增的 View 组件
  2. 需要替换的 Container 引用方式
  3. 提取出的公共 class / cva 变体
- 不要输出整页无关代码
- 不要生成未使用的样式
- 所有 className 保持简洁可维护

---

这套规则非常重要。  
**没有规则，AI 永远会朝“最快拼出来”的方向走，而不是“最可维护”的方向走。**

### 示例：BlockCardShell.tsx

```tsx
import { cva, type VariantProps } from "class-variance-authority";
import { Card } from "@/components/ui/card";
import { cn } from "@/lib/utils";

const blockCardVariants = cva(
  "transition-all rounded-xl border bg-card text-card-foreground shadow-sm",
  {
    variants: {
      state: {
        default: "border-border hover:shadow-md",
        selected: "border-primary ring-2 ring-primary/20 shadow-md",
        dragging: "border-primary/50 opacity-80 shadow-lg",
        editing: "border-primary bg-accent/30",
      },
      size: {
        sm: "p-3",
        md: "p-4",
      },
    },
    defaultVariants: {
      state: "default",
      size: "md",
    },
  }
);

type Props = React.ComponentProps<"div"> &
  VariantProps<typeof blockCardVariants>;

export function BlockCardShell({ className, state, size, ...props }: Props) {
  return <Card className={cn(blockCardVariants({ state, size }), className)} {...props} />;
}
```

使用：

```tsx
<BlockCardShell state={selected ? "selected" : "default"}>
  ...
</BlockCardShell>
```

这样比写一长串 className 干净很多。
