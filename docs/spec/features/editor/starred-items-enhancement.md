# 置顶功能增强

## 概述
在侧边栏置顶视图的基础上，增强了项目和文档的置顶功能，支持在资源管理器中直接置顶、拖拽排序和数量限制。

## 功能特性

### 1. 资源管理器星标按钮
- **项目星标**：在项目行 hover 时显示星标按钮，点击可置顶/取消置顶
- **文档星标**：在文档行 hover 时显示星标按钮，点击可置顶/取消置顶
- **视觉反馈**：已置顶的项目显示填充的金色星标图标

### 2. 拖拽排序
- **拖拽支持**：置顶项目支持拖拽重新排序
- **视觉反馈**：拖拽时项目半透明显示
- **持久化**：排序结果自动保存到 localStorage

### 3. 数量限制
- **最大数量**：最多置顶 10 个项目（项目+文档总数）
- **超限提示**：超过限制时弹出提示框
- **优先级**：用户可通过取消置顶旧项目来添加新项目

### 4. 错误处理
- **异步加载保护**：置顶视图加载项目名称时添加错误处理
- **后备方案**：加载失败时使用缓存的名称
- **空状态检查**：避免空数组触发不必要的加载

## 技术实现

### 数据结构
```typescript
interface StarredItem {
  id: string
  type: 'project' | 'document'
  name: string
  projectId?: string
  starredAt: Date
}
```

### 存储方案
- **存储位置**：`localStorage` (`blockos-starred-items`)
- **数据格式**：JSON 数组
- **同步机制**：全局 `toggleStar` 事件

### 组件交互
1. **ExplorerView**：提供星标按钮，触发置顶操作
2. **StarredView**：显示置顶列表，支持拖拽排序和取消置顶
3. **全局事件**：`toggleStar` 事件同步两个组件的状态

### 拖拽实现
- **HTML5 Drag and Drop API**
- **拖拽状态**：`draggedItem` 记录当前拖拽项
- **放置逻辑**：计算拖拽项和目标项的索引，重新排列数组

## 用户体验

### 交互流程
1. 用户在资源管理器中 hover 项目/文档
2. 显示星标按钮（未置顶时为空心，已置顶时为金色填充）
3. 点击星标按钮切换置顶状态
4. 置顶项目出现在侧边栏"置顶"视图中
5. 用户可拖拽置顶项目重新排序
6. 点击置顶项目的星标按钮取消置顶

### 视觉设计
- **星标颜色**：金色 (#f59e0b)
- **hover 效果**：星标按钮从透明到可见
- **拖拽效果**：拖拽时项目半透明 (opacity: 0.5)
- **主题适配**：支持 default 和 newsprint 主题

## 已知限制
- 置顶数量限制为 10 个
- 拖拽排序仅在置顶视图中可用
- 删除项目/文档时不会自动从置顶列表中移除（需要手动取消置顶）
