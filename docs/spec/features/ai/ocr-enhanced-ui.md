# OCR 增强 UI 功能需求

## 概述

本文档定义了 OCR 插件 UI/UX 增强功能的产品需求。该功能旨在将现有的单栏 OCR 界面升级为三栏布局，提供照片历史记录管理、图片预览和结果编辑等增强功能，显著提升用户体验和工作效率。

**完整技术需求文档**: [.kiro/specs/ocr-enhanced-ui/requirements.md](../../../../.kiro/specs/ocr-enhanced-ui/requirements.md)

## 背景

- 现有 OCR 插件提供基础的图片识别功能（拍照/上传 → 识别 → 插入/保存）
- 用户反馈需要更强大的历史记录管理和结果编辑能力
- 参考设计展示了三栏布局的优化方案

## 核心功能

### 1. 三栏布局界面

**左侧 - 历史记录列表（240px）**
- 显示所有上传/拍摄的照片历史
- 缩略图（80x80px）+ 时间戳 + 识别状态
- 按时间倒序排列（最新的在最上方）
- 支持滚动查看所有历史记录
- 点击选中照片，高亮显示

**中间 - 图片预览区域（50%）**
- 显示当前选中图片的完整预览
- 保持图片原始宽高比
- 支持图片缩放（适应容器大小）
- 底部显示"拍照"和"上传图片"按钮
- 支持摄像头拍照和文件上传

**右侧 - 识别结果编辑器（50%）**
- 显示 OCR 识别结果
- 支持文本编辑（textarea）
- 支持多行文本和格式保留
- 底部操作按钮：复制、插入编辑器、保存为 Block

### 2. 照片历史记录持久化

- 使用 IndexedDB 存储照片历史记录
- 数据库名称：`blockos-db`（复用现有数据库）
- 对象存储：`ocr-photos`
- 最多存储 100 条历史记录
- 超过限制时自动删除最旧的记录
- 支持删除单条历史记录

### 3. 响应式布局

- **桌面模式（≥768px）**：三栏布局（左中右）
- **移动模式（<768px）**：单栏布局 + 标签页切换
- 布局切换时保持当前选中状态
- 使用 CSS 媒体查询实现

### 4. 性能优化

- 虚拟滚动（历史记录超过 50 条时）
- 缩略图显示（80x80px）而非原图
- 图片懒加载
- 后台异步加载历史记录
- 限制单张图片 base64 大小不超过 5MB

### 5. 可访问性

- 键盘导航（上下箭头键选择历史记录）
- ARIA 标签（所有交互元素）
- 语义化 HTML 标签
- 对比度符合 WCAG AA 标准

## 用户流程

### 流程 1: 拍照识别

1. 用户打开 OCR 插件
2. 点击"拍照"按钮
3. 允许浏览器访问摄像头
4. 对准文字内容
5. 点击"拍摄"按钮
6. 照片自动添加到历史记录列表（顶部）
7. 中间区域显示照片预览
8. 点击"识别文字"按钮
9. 右侧显示识别结果
10. 编辑结果（可选）
11. 点击"插入编辑器"或"保存为 Block"

### 流程 2: 上传图片识别

1. 用户打开 OCR 插件
2. 点击"上传图片"按钮
3. 选择一张包含文字的图片
4. 照片自动添加到历史记录列表（顶部）
5. 中间区域显示照片预览
6. 点击"识别文字"按钮
7. 右侧显示识别结果
8. 编辑结果（可选）
9. 点击"插入编辑器"或"保存为 Block"

### 流程 3: 查看历史记录

1. 用户打开 OCR 插件
2. 左侧显示所有历史记录
3. 点击某条历史记录
4. 中间区域显示该照片预览
5. 右侧显示该照片的识别结果（如果已识别）
6. 可以重新识别或编辑结果

### 流程 4: 删除历史记录

1. 用户悬停在某条历史记录上
2. 显示删除按钮
3. 点击删除按钮
4. 显示确认对话框
5. 确认后从列表和数据库中删除
6. 如果删除的是当前选中项，清空预览和结果

## 技术约束

1. **插件系统架构约束**：
   - 必须完全兼容现有的插件系统架构（IPlugin 接口、Plugin_API、PluginRegistry）
   - 不得修改插件系统核心代码
   - 必须通过 Plugin_API 访问主应用功能

2. **技术栈约束**：
   - 必须使用 React 18 + TypeScript 5（严格模式）
   - 必须使用 IndexedDB 本地存储（数据库名称：blockos-db）
   - 必须使用 Bun 包管理器（不引入新的大型框架）

3. **代码质量约束**：
   - 所有函数参数和返回值必须有明确类型
   - 禁止循环依赖，依赖方向严格单向向下
   - 每个文件单一职责
   - 代码修改后必须通过 `bun run type-check`

4. **性能约束**：
   - 单张图片的 base64 大小不超过 5MB
   - 历史记录最多存储 100 条
   - 初始渲染时间不超过 100ms
   - IndexedDB 读写操作不超过 100ms

## 分阶段实施

### Phase 1: 基础三栏布局 + 历史记录列表

- 创建 OCREnhancedPanel 组件（三栏布局容器）
- 创建 HistoryList 组件（左侧历史记录列表）
- 实现 IndexedDB 存储（ocr-photos 对象存储）
- 实现历史记录持久化（最多 100 条）
- 实现历史记录排序（时间倒序）
- 实现历史记录选择和高亮
- 响应式布局（768px 断点）

### Phase 2: 图片预览 + 拍照/上传功能

- 创建 PreviewArea 组件（中间预览区域）
- 实现图片预览（保持宽高比、缩放适配）
- 实现拍照功能（摄像头 + 实时视频流）
- 实现上传功能（文件选择 + Base64 转换）
- 实现加载状态和错误处理
- 实现占位提示和空状态

### Phase 3: 识别结果编辑 + 操作功能

- 创建 ResultEditor 组件（右侧结果编辑器）
- 实现 OCR 识别触发（手动触发 + 重新识别）
- 实现识别结果显示（加载动画 + 错误提示）
- 实现识别结果编辑（textarea + 自动保存）
- 实现识别结果操作（复制 + 插入编辑器 + 保存为 Block）
- 实现历史记录删除（确认对话框 + 清空显示）
- 性能优化（虚拟滚动 + 缩略图 + 懒加载）
- 可访问性（键盘导航 + ARIA 标签）
