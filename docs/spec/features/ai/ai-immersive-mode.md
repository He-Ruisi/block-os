## 概述

为 BlockOS 添加 AI 沉浸式模式，提供全屏对话体验。用户可以在 AI 模式和混合模式之间无缝切换。

## 设计理念

**两种工作模式**：
- **AI 模式**：全屏沉浸式对话，聚焦 AI 交互
- **混合模式**：编辑器 + AI 面板，多任务协作

## 功能特性

### 1. AI 模式（AI Focus Mode）

#### 初始状态
- **全屏显示**：隐藏侧边栏和编辑器
- **问候语**：中央显示大号问候 "🤖 下午好，[用户名]"
- **巨大输入框**：居中显示，占据主要视觉焦点
- **模型选择**：输入框右下角显示当前模型
- **极简设计**：纯净背景，聚焦对话

#### 对话状态
- **输入框下沉**：用户输入后，输入框移动到底部
- **消息展开**：对话内容区域展开，显示消息列表
- **保持全屏**：仍然是 AI 模式，不显示编辑器
- **消息样式**：与混合模式完全相同

#### 切换到混合模式
- **触发条件**：用户点击"写入编辑器"按钮
- **自动创建文档**：创建新文档"AI 对话笔记"
- **自动切换**：布局自动切换到混合模式
- **编辑器展开**：左侧编辑器区域显示
- **内容写入**：AI 内容写入新文档
- **侧边栏隐藏**：默认隐藏侧边栏，聚焦编辑
- **平滑动画**：0.4s cubic-bezier 过渡

### 2. 混合模式（Hybrid Mode）

- **当前实现**：保持不变
- **三栏布局**：侧边栏 + 编辑器 + AI 面板
- **多任务**：可以同时编辑和对话

### 3. 模式切换

#### 切换触发
```
启动应用 → AI 模式
点击"写入编辑器" → 混合模式
点击"AI 模式"按钮 → AI 模式
```

#### 切换动画
- 平滑的布局过渡（0.3s ease）
- 组件淡入淡出
- 输入框位置动画

## 技术实现

### 状态管理

```typescript
type ViewMode = 'ai-focus' | 'hybrid'

interface AppState {
  viewMode: ViewMode
  hasMessages: boolean // 是否有对话消息
}
```

### 布局逻辑

```typescript
// AI 模式
if (viewMode === 'ai-focus') {
  // 隐藏 Sidebar、Editor
  // RightPanel 全屏
  // 输入框根据 hasMessages 决定位置
}

// 混合模式
if (viewMode === 'hybrid') {
  // 显示所有组件
  // 当前布局
}
```

### 组件结构

```
App.tsx
├── viewMode state
├── AI 模式
│   └── RightPanel (全屏)
│       ├── 问候语 (hasMessages === false)
│       ├── 大号输入框 (hasMessages === false)
│       ├── 消息列表 (hasMessages === true)
│       └── 底部输入框 (hasMessages === true)
└── 混合模式
    ├── ActivityBar
    ├── Sidebar
    ├── Editor
    └── RightPanel
```

## 交互流程

### 流程 1：纯对话
```
1. 启动应用（AI 模式）
2. 输入问题
3. AI 回复
4. 继续对话
5. （保持 AI 模式）
```

### 流程 2：对话 → 编辑
```
1. 启动应用（AI 模式）
2. 输入问题
3. AI 回复
4. 点击"写入编辑器"
5. 自动创建新文档"AI 对话笔记"
6. 切换到混合模式（平滑动画）
7. 内容写入新文档
8. 侧边栏默认隐藏
9. 可以编辑内容
```

### 流程 3：模式切换
```
1. 混合模式工作
2. 点击"AI 模式"按钮
3. 切换到 AI 模式
4. 聚焦对话
```
