## 关键变更说明

### Block 系统设计方案

#### 核心理念：混合式 Block 系统
- **隐式 Block**: 编辑器段落自动生成，临时性，文档内部使用
- **显式 Block**: 用户手动捕获，永久性，全局可引用

#### 设计对比
**Notion 模式**:
- 强制每个段落都是 Block
- 适合结构化文档

**Roam Research 模式**:
- 强制段落 Block + 双向链接
- 适合知识网络

**BlockOS 方案**:
- 隐式 Block（自动）+ 显式 Block（手动）
- 平衡写作流畅性和知识管理

#### 数据模型

**隐式 Block**:
```typescript
interface ImplicitBlock {
  id: string              // 临时 ID
  content: string
  type: 'paragraph' | 'heading' | 'list'
  documentId: string
  position: number
}
```

**显式 Block**:
```typescript
interface ExplicitBlock {
  id: string              // UUID
  content: string
  type: 'text' | 'heading' | 'list' | 'code' | 'ai-generated'
  source: {
    documentId?: string
    aiMessageId?: string
    capturedAt: Date
  }
  metadata: {
    title?: string
    tags: string[]
    createdAt: Date
    updatedAt: Date
  }
  links: {
    outgoing: string[]
    incoming: string[]
  }
}
```

#### 核心场景

**场景 1: 编辑器写作**
- 段落自动成为隐式 Block
- 选中重要段落 → Cmd+B → 捕获为显式 Block

**场景 2: AI 对话**
- AI 回复可直接捕获为显式 Block
- 带有 AI 来源标记

**场景 3: 选中文字发送给 AI**
- 选中内容作为上下文传递
- AI 可引用上下文回复

#### UI/UX 设计

**编辑器标识**:
- 隐式 Block: 悬停显示淡色边框 + `⋮⋮` 手柄
- 显式 Block: 左侧 `◆` 标记（紫色）

**捕获对话框**:
- 标题（可选）
- 标签管理
- 内容预览
- 取消/捕获按钮

**Block 空间面板**:
- 右侧面板新增 "Block 空间" 标签页
- 搜索、标签过滤
- Block 卡片列表

#### 技术实现

**存储**: IndexedDB + 索引（tags, createdAt, type）
**编辑器**: TipTap 自定义 Block 节点
**捕获**: 选中文字 API + Block 创建逻辑

#### 实现优先级

**Phase 2.5 (当前)**:
1. 选中文字发送给 AI
2. AI 回复捕获为 Block
3. Block 空间基础 UI

**Phase 3 (后续)**:
1. 隐式 Block 系统
2. 双向链接 `[[]]`

4. Block 关系图谱

### 设计优势
1. **不打断写作**: 隐式 Block 自动生成
2. **精准管理**: 显式 Block 手动捕获重要内容
3. **灵活引用**: 两种 Block 都可引用
4. **渐进实现**: 先显式后隐式，降低复杂度
