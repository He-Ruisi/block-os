## 11:37 - 解答 bun 命令不存在问题

### 完成内容
- 诊断 `bun run dev` 命令不存在的原因
- 提供两种解决方案

### 文件变更
- 无文件变更

### 关键说明
- **原因**：bun 安装在 `~/.bun/bin/bun`，但未加入 shell PATH
- **方案 1（推荐）**：永久修复，将 bun 加入 PATH：
  ```bash
  echo 'export PATH="$HOME/.bun/bin:$PATH"' >> ~/.zshrc
  source ~/.zshrc
  ```
- **方案 2**：每次使用完整路径 `~/.bun/bin/bun run dev`
- steering 文件 `bun-defaults.md` 中已记录完整路径备用

- 修复点击新文档后编辑区一片空白的问题

### 文件变更
- 修改：`src/components/Editor.tsx` - 修复文档加载逻辑
- 移动：`MULTI_PROJECT_QUICK_START.md` → `docs/guide/`
- 移动：`START_DEV_SERVER.md` → `docs/guide/`

### 关键说明
- **Bug 根因**：上次引入 documentId 支持时引入了两个问题：
  1. `if (doc.content)` 对空字符串 `""` 判断为 falsy，导致有内容的文档被当成新文档清空
  2. `editor` 实例变化时重复触发 useEffect，内容写入后又被覆盖
- **修复方案**：
  1. 改为 `doc.content && doc.content.trim() !== ''` 严格判断
  2. 添加 `loadedDocumentIdRef` 记录已加载的文档 ID，相同 ID 不重复加载
